# Документация проекта: Платформа Бронирования (Booking Platform)

Данный документ представляет собой полный технический отчет о проекте, созданный для учебного отчета (курсовой работы).

---

## 1. Обзор проекта
**Название проекта:** Booking Platform
**Тип приложения:** Веб-приложение для управления бронированием услуг (отели, спецтехника, мероприятия и т.д.).
**Основные функции:**
- Просмотр каталога услуг с фильтрацией и поиском.
- Детальная информация об услуге с картой (Yandex Maps).
- Система авторизации и регистрации пользователей.
- Личный кабинет пользователя с историей бронирований.
- Админ-панель для управления контентом (создание, редактирование, удаление услуг).
- Поддержка нескольких языков (Казахский, Русский, Английский).

---

## 2. Технологический стек

### Фронтенд (Frontend)
- **Библиотека:** React 19
- **Сборщик:** Vite
- **Стилизация:** Чистый CSS (Modern CSS) с использованием переменных и анимаций.
- **Иконки:** Lucide React
- **Управление состоянием:** React Context API (AuthContext, LanguageContext).
- **Библиотеки связи:** Axios (архитектура клиент-сервер через REST API).
- **Роутинг:** React Router DOM.
- **Карты:** Yandex Maps JS API 2.1.

### Бэкенд (Backend)
- **Среда выполнения:** Node.js
- **Фреймворк:** Express.js
- **Безопасность:** JWT (JSON Web Tokens) для авторизации, Bcrypt для хеширования паролей.
- **Загрузка файлов:** Multer (хранение изображений на сервере).
- **База данных:** PostgreSQL.
- **Драйвер БД:** `pg` (node-postgres).

## 3. Архитектура базы данных

База данных состоит из 4 основных таблиц (плюс таблица категорий):

### Таблица `users` (Пользователи)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | SERIAL (PK) | Уникальный идентификатор |
| `name` | TEXT | Имя пользователя |
| `email` | TEXT | Электронная почта (Unique) |
| `password_hash`| TEXT | Хеш пароля |
| `is_admin` | BOOLEAN | Флаг администратора |

### Таблица `services` (Услуги)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | SERIAL (PK) | Уникальный идентификатор |
| `title` | VARCHAR(255)| Название услуги |
| `price` | NUMERIC | Стоимость |
| `images` | TEXT[] | Массив ссылок на изображения |
| `address` | TEXT | Физический адрес |
| `category_id` | INTEGER (FK) | Ссылка на категорию |
| `latitude/long`| NUMERIC | Координаты для карты |

### Таблица `bookings` (Бронирования)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | SERIAL (PK) | Уникальный идентификатор |
| `user_id` | INTEGER (FK) | Кто забронировал |
| `service_id` | INTEGER (FK) | Какая услуга |
| `status` | VARCHAR | Статус (pending, paid, cancelled) |

### Таблица `categories` (Категории)
- Хранит ключи для перевода категорий (`cars`, `housing`, `events` и т.д.).

## 4. Как все связано (Архитектура связи)

1. **Клиент -> Сервер:** Фронтенд на React отправляет HTTP-запросы (GET, POST, PUT, DELETE) через Axios на Node.js сервер.
2. **Авторизация:** При логине сервер выдает JWT, который сохраняется в `localStorage`. Все защищенные запросы (например, в админ-панель) отправляются с этим токеном в заголовке `Authorization`.
3. **Хранение данных:** Вся динамическая информация хранится в PostgreSQL. Для работы с изображениями используется middleware `multer`, который сохраняет файлы в папку `backend/uploads/`.
4. **Локализация:** `LanguageContext` на фронтенде перехватывает все текстовые ключи и сопоставляет их с объектом переводов в зависимости от выбранного языка (`kk`, `ru`, `en`).

## 5. Инструкция по запуску

### Бэкенд:
1. Перейти в `cd backend`.
2. Установить зависимости: `npm install`.
3. Настроить файл `.env` (DATABASE_URL, JWT_SECRET).
4. Запустить: `node server.js`.

### Фронтенд:
1. Перейти в `cd frontend`.
2. Установить зависимости: `npm install`.
3. Настроить `VITE_API_URL` в `.env`.
4. Запустить: `npm run dev`.
